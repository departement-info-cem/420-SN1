        :::info Qu'est-ce qu'une expression booléenne?

        Une expression booléenne est une expression qui évalue à une valeur de vérité ([`bool`](/aidememoire/types/)), c'est-à-dire soit `True` (vrai) soit `False` (faux).<br />
        Elles sont utilisées pour prendre des décisions dans le code, comme dans les instructions conditionnelles (`if`, `elif`, `else`) et les boucles (`while`, `for`).

        :::

        ---

        :::note Opérateurs de comparaison

        Les opérateurs de comparaison sont utilisés pour comparer deux valeurs.<br />
        Voici les principaux opérateurs de comparaison en Python :

        <AideMemoireExpressionsBoolennes />

        :::

        ---

        :::tip Exemples d'utilisation
        ```python showLineNumbers
        a = 5
        b = 10
        print(a < b)   # Affiche True
        print(a == b)  # Affiche False
        print(a + 5 >= b - max(1, min(4, a, 6), 5))   # Affiche True, explications ci-dessous!
        ```

Les 2 opérandes d’un opérateur de comparaison peuvent être le résultat de l'évaluation d’expressions. \
Python évalue l’expression selon la [priorité des opérateurs](/aidememoire/operateurs?onglet=priorite), en remplaçant chaque sous-expression par sa valeur résultante.

Voici comment Python évalue l’instruction #5, étape par étape :

        ```python 
        print(a + 5 >= b - max(1, min(4, a, 6), 5))  # Instruction originale
        print(5 + 5 >= b - max(1, min(4, 5, 6), 5))  # Remplace 'a' par sa valeur
        print(10 >= b - max(1, min(4, 5, 6), 5))     # Calcule la somme de '5 + 5'
        print(10 >= 10 - max(1, min(4, 5, 6), 5))    # Remplace 'b' par sa valeur
        print(10 >= 10 - max(1, 4, 5))               # La fonction min(...) doit être résolue avant la fonction max(...)
        print(10 >= 10 - 5)                          # La fonction max(...) est résolue
        print(10 >= 5)                               # L'opération >=
        print(True)                                  # Affiche "True" dans la console
        ```
        :::

        ---

        :::note Opérateurs logiques

        Les opérateurs logiques sont utilisés pour combiner plusieurs expressions booléennes. \
        Ils sont inclus dans le tableau des expressions booléennes ci-dessus (`and`, `or`, `not`).

        :::

        ---

        :::info Priorité des opérateurs
        La priorité des opérateurs détermine l'ordre dans lequel les opérations sont effectuées dans une expression.<br />
        En Python, la priorité des opérateurs est la suivante (de la plus haute à la plus basse) :
        1. Parenthèses `()`
        2. Opérateurs de comparaison (`==`, `!=`, `<`, `<=`, `>`, `>=`)
        3. Opérateur `not`
        4. Opérateur `and`
        5. Opérateur `or`

        Un ordre plus exhaustif des priorités des opérateurs est disponible [ici dans l'aide-mémoire](/aidememoire/operateurs?onglet=priorite).
        :::

        ---

        :::tip Exemple de priorité
        ```python showLineNumbers
        a = 5
        b = 10
        c = 15
        print(a < b and b < c)  # Évalué comme (a < b) and (b < c), affiche True
        print(a < b or b > c)   # Évalué comme (a < b) or (b > c), affiche True
        print(not a < b)        # Évalué comme not (a < b), affiche False

        # On peut utiliser des parenthèses pour modifier l'ordre d'évaluation :
        print(a < b or b > c and a == 5)  # Évalué comme a < b or (b > c and a == 5), affiche True
        print((a < b or b > c) and a == 5) # Évalué comme (a < b or b > c) and a == 5, affiche True
        ```
        :::

        ---

        :::info Stockage dans des variables
        On peut stocker le résultat d'une expression booléenne dans une variable pour une utilisation ultérieure.
        ```python showLineNumbers
        a = 5
        b = 10
        est_plus_petit = a < b  # est_plus_petit sera True
        est_egal = a == b       # est_egal sera False
        print(est_plus_petit)    # Affiche True
        print(est_egal)          # Affiche False
        ```
        :::

    <TabItem value="if" label="if">

        :::info On veut pas le sawouère, on veut le wouère
        C'est bien beau, ça, les expressions booléennes, mais à quoi ça sert?<br />
        👉 Elles servent principalement à contrôler le flux d'exécution du programme, \
        c'est-à-dire à décider quelles parties du code doivent être exécutées en fonction de certaines conditions.
        :::

        ---

        :::tip Structure de base d'une instruction conditionnelle
        Voici un exemple simple d'instruction conditionnelle en Python :
        ```python showLineNumbers
        age = 18
        if age >= 18 :
            print("Vous êtes majeur.")
        ```
        Dans cet exemple, l'instruction `if` vérifie si la variable `age` est supérieure ou égale à 18.<br />
        Si c'est le cas (si la condition est `True`), le code indenté sous l'instruction `if` est exécuté, affichant "Vous êtes majeur.".<br />
        Si la condition est `False`, le code indenté n'est pas exécuté.
        :::

        ---

        :::warning Indentation
        En Python, l'indentation (les espaces ou tabulations avant une ligne de code) est cruciale pour définir les blocs de code.<br />
        Le code qui doit être exécuté conditionnellement doit être indenté sous l'instruction `if`.<br />
        Par exemple :
        ```python showLineNumbers
        if condition :
            # Code indenté, exécuté si la condition est True
            faire_quelque_chose_conditionnel()
            faire_autre_chose_conditionnel()
        # Code non indenté, exécuté indépendamment de la condition
        faire_autre_chose_en_tout_temps()
        ```
        Exemple d'erreur courante due à une mauvaise indentation :
        ```python showLineNumbers
        if condition :
        faire_quelque_chose()  # Erreur : cette ligne doit être indentée
        ```
        Exemple concret :
        ```python showLineNumbers
        age = 20
        if age >= 18 :
            print("Vous êtes majeur.")
            print("Vous avez le droit de vote!")
            print("Un grand pouvoir implique de grandes responsabilités.")
        print("Bienvenue!")  # Toujours exécuté, indépendamment de l'âge
        ```
        :::

    <TabItem value="elif" label="elif">

        :::info Un elfe?
        Non!<br />
        `elif` est la contraction de "else if", qui signifie "sinon si".<br />
        Elle permet de vérifier une autre condition si la première condition dans l'instruction `if` est `False`.

        Par exemple :
        ```python showLineNumbers
        age = 16
        if age >= 18 :
            print("Vous êtes majeur.")
        elif age >= 13 :
            print("Vous êtes adolescent.")
        ```
        :::

        ---

        :::note Multiple elif
        On peut avoir plusieurs clauses `elif` pour vérifier plusieurs conditions différentes.<br />
        Par exemple :
        ```python showLineNumbers
        age = 10
        if age >= 18 :
            print("Vous êtes majeur.")
        elif age >= 13 :
            print("Vous êtes adolescent.")
        elif age >= 3 :
            print("Vous êtes un enfant.")
        ```
        :::

        ---

        :::tip Ordre des conditions
        L'ordre des conditions est important.<br />
        Les conditions sont évaluées de haut en bas, et la première condition qui est `True` déclenche l'exécution du bloc de code correspondant.<br />
        Par exemple, dans le code suivant, la condition `age >= 13` sera toujours `True` pour tous les adultes, et la condition `age >= 18` ne sera jamais atteinte :
        ```python showLineNumbers
        age = 21
        if age >= 13 :
            print("Vous êtes adolescent.")
        elif age >= 18 :
            print("Vous êtes majeur.")
        ```
        Pour éviter cela, il faut toujours commencer par la condition la plus restrictive (la plus élevée) et descendre vers les conditions moins restrictives :
        ```python showLineNumbers
        age = 21
        if age >= 18 :
            print("Vous êtes majeur.")
        elif age >= 13 :
            print("Vous êtes adolescent.")
        ```
        :::

        ---

        :::warning Pas de `elif` sans `if`
        Une clause `elif` doit toujours être précédée d'une instruction `if`.<br />
        Par exemple, le code suivant est incorrect et provoquera une erreur de syntaxe :
        ```python showLineNumbers
        age = 16
        elif age >= 13 :  # Erreur : 'elif' sans 'if'
            print("Vous êtes adolescent.")
        ```
        :::

    <TabItem value="else" label="else">

        :::info Le sinon
        `else` signifie "sinon".<br />
        Elle permet de définir un bloc de code qui sera exécuté si toutes les conditions précédentes (`if` et `elif`) sont `False`.

        Par exemple :
        ```python showLineNumbers
        age = 10
        if age >= 18 :
            print("Vous êtes majeur.")
        elif age >= 13 :
            print("Vous êtes adolescent.")
        else :
            print("Vous êtes un enfant.")
        ```
        Dans cet exemple, si `age` est inférieur à 13, le message "Vous êtes un enfant." sera affiché.
        :::

        ---

        :::tip Utilisation avec if et elif
        La clause `else` est souvent utilisée à la fin d'une série d'instructions `if` et `elif` pour capturer tous les cas non couverts par les conditions précédentes.<br />
        Par exemple :
        ```python showLineNumbers
        score = 85
        if score >= 90 :
            print("Excellent")
        elif score >= 75 :
            print("Bien")
        elif score >= 50 :
            print("Passable")
        else :
            print("Échec")
        ```
        :::

        ---

        :::warning Pas de condition avec else
        Contrairement à `if` et `elif`, la clause `else` ne prend pas de condition.<br />
        Elle est simplement exécutée si toutes les conditions précédentes sont `False`. Par exemple, le code suivant est incorrect et provoquera une erreur de syntaxe :
        ```python showLineNumbers
        score = 85
        if score >= 90 :
            print("Excellent")
        elif score >= 75 :
            print("Bien")
        elif score >= 50 :
            print("Passable")
        else score < 50 :  # Erreur : 'else' ne doit pas avoir de condition
            print("Échec")
        ```
        :::

        ---

        :::tip else sans elif
        Il est possible d'utiliser `else` directement après un `if`, sans avoir besoin de `elif`.<br />
        Par exemple :
        ```python showLineNumbers
        age = 20
        if age >= 18 :
            print("Vous êtes majeur.")
        else :
            print("Vous êtes mineur.")
        ```
        :::

        :::info Intégration avec les fonctions et les listes

        Les structures conditionnelles (`if/elif/else`, `match/case`) peuvent être utilisées à l'intérieur de fonctions.

        **Exemple 1 — `if/else` simple :**
        ```python showLineNumbers
        def est_pair(n):
            if n % 2 == 0:
                return True  # Faire modulo 2 sur un nombre pair donne toujours 0!
            else:
                return False
        
        a = 33
        b = 52
        print(f"{a} est un nombre pair : {est_pair(a)}")
        print(f"{b} est un nombre pair : {est_pair(b)}")
        ```

        ```
        33 est un nombre pair : False
        52 est un nombre pair : True
        ```
        **Exemple 2 — `if/else` avec paramètre optionnel et calcul par palier :**

        La fonction suivante calcule la paie brute d'un employé. Elle accepte un troisième paramètre optionnel pour activer le calcul des heures supplémentaires au taux double.

        ```python showLineNumbers
        def calculer_paie_brute(nb_heures, taux_horaire, temps_double_apres_40_heures=False):
            salaire = nb_heures * taux_horaire
            if temps_double_apres_40_heures and nb_heures >= 40:
                nb_heures_temps_supp = nb_heures - 40
                salaire = salaire + nb_heures_temps_supp * taux_horaire * 2
            return salaire

        nb_heures = 45
        taux_horaire = 20
        print(f"Calcul de la paie brute pour {nb_heures} heures à un taux horaire de {taux_horaire}$/h.")
        print(f"  Avec le temps > 40 payé au taux doublé   : {calculer_paie_brute(nb_heures, taux_horaire, True)}$")
        print(f"  Avec le temps > 40 payé au taux régulier : {calculer_paie_brute(nb_heures, taux_horaire)}$")
        ```

        ```
        Calcul de la paie brute pour 45 heures à un taux horaire de 20$/h.
          Avec le temps > 40 payé au taux doublé   : 1100$
          Avec le temps > 40 payé au taux régulier : 900$
        ```

        **Exemple 3 — `if/elif/else` avec trois listes parallèles :**

        On peut stocker plusieurs valeurs dans des **listes parallèles** et appeler notre fonction sur chaque élément.
        Voici une liste d'éléments du tableau périodique avec leur masse atomique (en u.m.a.) et leur nombre de protons.
        On veut **classifier la stabilité** de chaque noyau en fonction du rapport entre sa masse atomique et son nombre de protons — un indicateur simplifié du ratio neutrons/protons :

        ```python showLineNumbers
        def stabilite_noyau(masse_atomique, nb_protons):
            """
            Classifie la stabilité d'un noyau selon le ratio neutrons/protons.

            Paramètres :
            masse_atomique (int) : Nombre de masse de l'élément (protons + neutrons).
            nb_protons (int)     : Nombre de protons dans le noyau.

            Retourne :
            str : "stable", "légèrement instable" ou "très instable".
            """
            nb_neutrons = masse_atomique - nb_protons
            ratio = nb_neutrons / nb_protons
            if ratio <= 1.1 :
                return "stable"
            elif ratio <= 1.5 :
                return "légèrement instable"
            else :
                return "très instable"

        masses_atomiques = [1,4,7,12,14,16]
        symboles = ["H","He","Li","C","N","O"]
        nb_protons = [1,2,3,6,7,8]

        print(f"{symboles[0]} (masse {masses_atomiques[0]}) - stabilité : {stabilite_noyau(masses_atomiques[0], nb_protons[0])}")
        print(f"{symboles[1]} (masse {masses_atomiques[1]}) - stabilité : {stabilite_noyau(masses_atomiques[1], nb_protons[1])}")
        print(f"{symboles[2]} (masse {masses_atomiques[2]}) - stabilité : {stabilite_noyau(masses_atomiques[2], nb_protons[2])}")
        print(f"{symboles[3]} (masse {masses_atomiques[3]}) - stabilité : {stabilite_noyau(masses_atomiques[3], nb_protons[3])}")
        print(f"{symboles[4]} (masse {masses_atomiques[4]}) - stabilité : {stabilite_noyau(masses_atomiques[4], nb_protons[4])}")
        print(f"{symboles[5]} (masse {masses_atomiques[5]}) - stabilité : {stabilite_noyau(masses_atomiques[5], nb_protons[5])}")
        ```

        ```
        H (masse 1) - stabilité : stable
        He (masse 4) - stabilité : stable
        Li (masse 7) - stabilité : légèrement instable
        C (masse 12) - stabilité : stable
        N (masse 14) - stabilité : stable
        O (masse 16) - stabilité : stable
        ```

        👉 On remarque qu'on répète 6 fois la même instruction `print` en changeant seulement l'index. \
        C'est exactement le genre de répétition qu'une **boucle `for`** éliminera en quelques lignes, vous verrez ça bientôt!

    <TabItem value="comparaison" label="Comparaison">


        :::danger ⚠️ `if if if` vs `if elif else` — ce n'est pas la même chose!

        C'est l'une des erreurs les plus fréquentes chez les débutants : \
        utiliser plusieurs `if` indépendants là où un `if/elif/else` serait nécessaire. \
        La différence est fondamentale et peut avoir des impacts sur les performances.

        Prenons cet exemple avec `score = 95` :

        ```python showLineNumbers
        # Version avec des if if if
        score = 95
        if score >= 90 :
            print("Excellent")  # ✅ Exécuté (95 >= 90 est True)
        if score >= 75 :
            print("Bien")       # ✅ Exécuté (95 >= 75 est True)
        if score >= 50 :
            print("Passable")   # ✅ Exécuté (95 >= 50 est True)
        else :
            print("Échec")      # ⏭️ Bloc ignoré (car la condition score >= 50 a donné True)
        ```

        Résultat affiché :
        ```
        Excellent
        Bien
        Passable
        ```
        👉 Avec des `if` indépendants, Python évalue **chaque condition séparément**, de haut en bas, **sans jamais s'arrêter**. Comme les 3 conditions sont `True` pour `score = 95`, les **3 blocs sont exécutés**. Ce n'est presque certainement pas ce qu'on voulait! Il aurait été préférable que Python arrête d’évaluer les autres conditions dès qu’il trouve une condition donnant `True`.

        :::

        ---

        :::tip ✅ La bonne approche : `if elif else`
        ```python showLineNumbers
        # Version avec if elif else
        score = 95
        if score >= 90 :
            print("Excellent")  # ✅ Exécuté (95 >= 90 est True) → Python arrête ici
        elif score >= 75 :
            print("Bien")       # ⏭️ Bloc ignoré (car une condition d'un bloc précédent a donné True)
        elif score >= 50 :
            print("Passable")   # ⏭️ Bloc ignoré (car une condition d'un bloc précédent a donné True)
        else :
            print("Échec")      # ⏭️ Bloc ignoré (car une condition d'un bloc précédent a donné True)
        ```

        Résultat affiché :
        ```
        Excellent
        ```

        👉 Avec `if/elif/else`, dès qu'une condition est `True`, Python exécute le bloc correspondant et **saute immédiatement le reste** : les autres conditions ne sont **jamais évaluées**. Un seul bloc s'exécute, peu importe le nombre de `elif`. Python a donc exécuté uniquement le bloc associé à `score >= 90`.
        :::

        ---

        :::note 🧠 Comment choisir entre les 2 approches

        | Situation | Utiliser |
        |-----------|----------|
        | Les cas sont **mutuellement exclusifs** (un seul doit s'exécuter) | <C><K>if</K> &#47; <K>elif</K> &#47; <K>else</K></C> |
        | Les vérifications sont **indépendantes** les unes des autres | Plusieurs <C><K>if</K></C> séparés |

        **Exemple où plusieurs `if` séparés sont appropriés** :
        ```python showLineNumbers
        ph = 3.5
        temperature_celsius = 85
        pression_atm = 1.8

        if ph < 6 :
            print("⚠️ Solution acide : portez des gants de protection.")       # ✅ Exécuté
        if temperature_celsius > 60 :
            print("🌡️ Température élevée : risque de brûlure.")                # ✅ Exécuté
        if pression_atm > 1.5 :
            print("💨 Pression élevée : vérifiez l'étanchéité du système.")    # ✅ Exécuté
        ```
        Ici, les 3 avertissements sont **complémentaires** et ont du sens ensemble. Plusieurs `if` indépendants sont donc le bon choix.

        :::

        ---

        :::info Récapitulatif if, elif, else

        Pour récapituler, la structure d'une instruction conditionnelle en Python est la suivante :
        1. `if` : Obligatoire pour commencer une instruction conditionnelle.
        2. `elif` : Les blocs `elif` sont optionnels.
            - Il peut y en avoir 0, 1, 2 ou plus.
            - Chaque `elif` vérifie une condition, seulement si toutes les conditions précédentes sont `False`.
        3. `else` : Le bloc `else` est optionnel.
            - Il ne peut y avoir qu’un seul `else`.
            - Il est exécuté si toutes les conditions précédentes sont `False`.
            - Le bloc `else` ne prend pas de condition.

        :::

        ---

        :::tip Autres points importants à retenir

        - **Indentation** : le code à exécuter conditionnellement doit être indenté sous l'instruction correspondante.
        - **Ordre des conditions** :
            - les conditions sont évaluées de haut en bas;
            - la première condition `True` déclenche l'exécution du bloc de code correspondant;
            - dans une structure `if/elif/else`, les autres conditions sont ignorées dès qu’une condition est `True`.
        :::

    <TabItem value="imbrication" label="Imbrication">

        :::info Des conditions dans des conditions!
        Il est tout à fait possible d'écrire une structure conditionnelle **à l'intérieur** d'une autre.\
        On appelle cela des structures **imbriquées** (*nested* en anglais).

        C'est utile quand une décision dépend d'une autre, comme dans la vraie vie!
        :::

        :::tip Exemple de base — classer un échantillon
        Imaginons qu'on veuille classer un échantillon selon son pH **ET** sa température :

        ```python showLineNumbers
        ph = 4.2
        temperature_celsius = 75

        if ph < 7 :
            print("Solution acide.")
            if temperature_celsius > 60 :               # Condition imbriquée dans le if
                print("⚠️ Acide ET chaude : danger!")
            else :
                print("Acide mais à température normale.")
        else :
            print("Solution neutre ou basique.")
            if temperature_celsius > 60 :               # Condition imbriquée dans le else
                print("⚠️ Solution chaude : manipuler avec précaution.")
            else :
                print("Conditions stables.")
        ```

        Résultat affiché avec `ph = 4.2` et `temperature_celsius = 75` :
        ```
        Solution acide.
        ⚠️ Acide ET chaude : danger!
        ```

        👉 Python entre d'abord dans le bloc `if ph < 7`, puis évalue la condition sur la température **seulement à l'intérieur de ce bloc**.
        :::

        :::note L'indentation, encore plus importante avec l'imbrication!
        Chaque niveau d'imbrication ajoute **un niveau d'indentation supplémentaire**.\
        C'est ce qui permet à Python de savoir à quel bloc appartient chaque instruction.

        ```python showLineNumbers
        if condition_1 :
            # Bloc de la condition 1 (indenté 1 fois)
            if condition_2 :
                # Bloc de la condition 2 (indenté 2 fois)
                faire_quelque_chose()
            else :
                # Bloc else de la condition 2 (indenté 2 fois)
                faire_autre_chose()
        else :
            # Bloc else de la condition 1 (indenté 1 fois)
            faire_encore_autre_chose()
        ```

        :::

        :::tip Exemple concret — classification d'une étoile 🌟
        En astronomie, on peut classer une étoile selon sa **luminosité** et sa **température de surface** :

        ```python showLineNumbers
        luminosite = 500        # En unités solaires
        temperature_K = 8000    # En kelvins

        if luminosite >= 100 :
            print("Étoile très lumineuse.")
            if temperature_K >= 10000 :
                print("⭐ Classification : étoile bleue géante.")
            elif temperature_K >= 6000 :
                print("⭐ Classification : étoile blanche géante.")
            else :
                print("⭐ Classification : supergéante froide.")
        elif luminosite >= 1 :
            print("Étoile de luminosité moyenne.")
            if temperature_K >= 6000 :
                print("⭐ Classification : étoile de la séquence principale (type solaire ou plus chaud).")
            else :
                print("⭐ Classification : étoile naine froide.")
        else :
            print("Étoile peu lumineuse.")
            if temperature_K >= 4000 :
                print("⭐ Classification : naine blanche.")
            else :
                print("⭐ Classification : naine brune.")
        ```

        Résultat affiché avec `luminosite = 500` et `temperature_K = 8000` :
        ```
        Étoile très lumineuse.
        ⭐ Classification : étoile blanche géante.
        ```

        👉 Le premier `if/elif/else` traite la luminosité, puis, à l'intérieur de **chacun** de ses blocs, un second `if/elif/else` affine la classification selon la température.
        :::

        :::warning Attention à la profondeur d'imbrication!
        Plus on imbrique de niveaux, plus le code devient difficile à lire et à déboguer.\
        En général, une bonne règle de base est de ne pas dépasser **2 à 3 niveaux**.\
        Si vous vous retrouvez avec plus de 3 niveaux, c'est souvent le signe que le code pourrait être réorganisé, par exemple en utilisant des fonctions.

        ```python showLineNumbers
        # ❌ Difficile à lire — trop de niveaux d'imbrication
        if condition_1 :
            if condition_2 :
                if condition_3 :
                    if condition_4 :
                        faire_quelque_chose()  # On s'y perd!
        ```

        ```python showLineNumbers
        # ✅ Plus lisible — on peut souvent combiner les conditions avec and
        if condition_1 and condition_2 and condition_3 and condition_4 :
            faire_quelque_chose()
        ```
        :::

        :::note 🧠 Quand utiliser l'imbrication?

        | Situation | Approche recommandée |
        |-----------|----------------------|
        | La 2e condition ne s'évalue **que si** la 1re est vraie | **Imbrication** |
        | Les conditions sont **indépendantes** les unes des autres | Plusieurs <C><K>if</K></C> séparés |
        | Les conditions peuvent être combinées simplement | <C><K>and</K></C> / <C><K>or</K></C> dans un seul <C><K>if</K></C> |

        👉 L'imbrication est le bon choix quand la **logique est hiérarchique** : on vérifie d'abord une catégorie générale, puis on affine à l'intérieur de cette catégorie.
        :::

    <TabItem value="DRY" label="DRY">

        :::info Éviter la répétition : les structures conditionnelles
        En programmation, on suit souvent le principe **DRY** (*Don't Repeat Yourself*). \
        Si vous remarquez que vous écrivez **la même instruction plusieurs fois** dans des blocs différents, c'est le signe qu'elle n'est pas vraiment conditionnelle! <br/>

        Si une instruction se retrouve à la fois dans le `if` **et** dans le `else`, elle s'exécuterait dans **tous les cas** de toute façon. \
        Elle devrait donc être placée **en dehors** du bloc conditionnel — soit **avant**, soit **après**, selon sa position dans les blocs de code.
        :::

        ---

        :::danger ⚠️ Avant le bloc — exemple avec une instruction redondante au début
        Repérez l'instruction répétée inutilement dans les deux blocs :
        ```python showLineNumbers
        # Version redondante
        temperature = 37.8

        if temperature > 37.5 :
            print("Initialisation du protocole de mesure...")  # ❌ Répété au début des 2 blocs!
            print("⚠️ Fièvre détectée : consulter un médecin.")
        else :
            print("Initialisation du protocole de mesure...")  # ❌ Répété au début des 2 blocs!
            print("✅ Température normale.")
        ```
        Peu importe la valeur de `temperature`, l'initialisation du protocole est **toujours** affichée.\
        La répéter dans les deux blocs est inutile et rend le code plus difficile à maintenir.
        :::

        ---

        :::tip ✅ La bonne approche : sortir l'instruction commune avant le bloc
        ```python showLineNumbers
        # Version sans redondance
        temperature = 37.8

        print("Initialisation du protocole de mesure...")  # ✅ Exécuté dans tous les cas avant les 2 blocs

        if temperature > 37.5 :
            print("⚠️ Fièvre détectée : consulter un médecin.")
        else :
            print("✅ Température normale.")
        ```
        Le code est plus court, plus lisible, et plus facile à modifier : si le message d'initialisation change, on n'a qu'**un seul endroit** à modifier.
        :::

        ---

        :::danger ⚠️ Après le bloc — exemple avec une instruction redondante à la fin
        Même problème, mais cette fois l'instruction répétée se trouve à la **fin** de chaque bloc :
        ```python showLineNumbers
        # Version redondante
        masse_g = 5.2
        volume_mL = 10.0

        if masse_g > 0 and volume_mL > 0 :
            densite = masse_g / volume_mL
            print(f"Densité calculée : {densite} g/mL")
            print("--- Fin de l'analyse ---")  # ❌ Répété à la fin des 2 blocs!
        else :
            print("⚠️ Paramètres invalides, calcul impossible.")
            print("--- Fin de l'analyse ---")  # ❌ Répété à la fin des 2 blocs!
        ```
        :::

        ---

        :::tip ✅ La bonne approche : sortir l'instruction commune après le bloc
        ```python showLineNumbers
        # Version sans redondance
        masse_g = 5.2
        volume_mL = 10.0

        if masse_g > 0 and volume_mL > 0 :
            densite = masse_g / volume_mL
            print(f"Densité calculée : {densite} g/mL")
        else :
            print("⚠️ Paramètres invalides, calcul impossible.")

        print("--- Fin de l'analyse ---")  # ✅ Exécuté dans tous les cas après les 2 blocs
        ```
        :::

        ---

        :::note 🧠 Récapitulatif — où placer les instructions communes?

        | Position de la redondance | Solution |
        |---------------------------|----------|
        | Au **début** de chaque bloc | Placer l'instruction **avant** le bloc `if` |
        | À la **fin** de chaque bloc | Placer l'instruction **après** le bloc `if/else` |
        | Au **début et à la fin** | Placer une instruction **avant** et une **après** |

        👉 La règle est simple : si une instruction s'exécute **dans tous les cas**, elle ne devrait **jamais** se retrouver à l'intérieur d'un `if` ou d'un `else`.
        :::

        ---

        :::tip Exemple complet — redondance au début ET à la fin
        ```python showLineNumbers
        # Version redondante
        pression_atm = 2.3

        if pression_atm > 2.0 :
            print("=== Rapport de pression ===")     # ❌ Début redondant
            print("⚠️ Pression critique détectée!")
            print("=== Fin du rapport ===")          # ❌ Fin redondante
        else :
            print("=== Rapport de pression ===")     # ❌ Début redondant
            print("✅ Pression dans les normes.")
            print("=== Fin du rapport ===")          # ❌ Fin redondante
        ```

        ```python showLineNumbers
        # Version sans redondance
        pression_atm = 2.3

        print("=== Rapport de pression ===")            # ✅ Redondance déplacée avant les blocs

        if pression_atm > 2.0 :
            print("⚠️ Pression critique détectée!")
        else :
            print("✅ Pression dans les normes.")

        print("=== Fin du rapport ===")                 # ✅ Redondance déplacée après les blocs
        ```
        Le résultat affiché est **identique**, mais la version propre est bien plus facile à lire et à maintenir!
        :::

        ---

        :::warning Attention — ne pas confondre redondance et répétition intentionnelle!
        Parfois, deux instructions se ressemblent mais **ne sont pas identiques** : elles ne peuvent donc pas être sorties du bloc.
        ```python showLineNumbers
        # ✅ Ce n'est PAS de la redondance, car les messages sont différents!
        ph = 4.2

        if ph < 7 :
            print(f"Le pH de {ph} indique une solution acide.")
        else :
            print(f"Le pH de {ph} indique une solution basique ou neutre.")
        ```
        Ici, les deux `print` contiennent du texte différent. Ils doivent rester dans leurs blocs respectifs.
        :::

    </TabItem>

    <TabItem value="compacte" label="Forme compacte">

        :::info Forme compacte

        Pour les instructions conditionnelles simples, on peut utiliser une forme compacte en une seule ligne.<br />
        Par exemple :
        ```python showLineNumbers
        age = 20
        print("Vous êtes majeur.") if age >= 18 else print("Vous êtes mineur.")
        ```

        On peut aussi utiliser cette forme compacte pour stocker le résultat dans une variable :
        ```python showLineNumbers
        age = 20
        statut = "majeur" if age >= 18 else "mineur"
        print(statut)  # Affiche "majeur"
        ```
        :::

        ---

        :::tip Limites de la forme compacte

        La forme compacte est utile pour des conditions simples et des actions courtes.<br />
        Cependant, pour des conditions plus complexes ou des blocs de code plus longs, <br />
        il est préférable d'utiliser la forme standard avec indentation pour une meilleure lisibilité.
        :::

        ---

        :::warning Avec des elif
        On peut aussi utiliser des `elif` dans une forme compacte, mais cela peut rapidement devenir difficile à lire.<br />
        Par exemple :
        ```python showLineNumbers
        score = 85
        resultat = "Excellent" if score >= 90 else "Bien" if score >= 75 else "Passable" if score >= 50 else "Échec"
        print(resultat)  # Affiche "Bien"
        ```
        Si on a besoin de `elif`, il est préférable d'utiliser la forme standard pour une meilleure lisibilité.
        :::

    <TabItem value="exceptions-validation" label="Validation raise/try/except">

        :::info Pourquoi valider les paramètres?
        Quand on écrit une fonction, il faut s'assurer que les **entrées** (paramètres) ont du sens.\
        Sinon, on risque d'obtenir des résultats absurdes ou des erreurs difficiles à comprendre.

        Comparez ces deux versions :

        ```python showLineNumbers
        # Version 1 – sans validation
        def densite(masse, volume):
            return masse / volume

        print(densite(10, 2))   # ✅ 5.0
        print(densite(-3, 5))   # ⚠️ -0.6 g/mL sans avertissement (résultat absurde), le programme continue malgré tout!
        print(densite(8, 0))    # ❌ Erreur Python brute (ZeroDivisionError), le programme arrête ici!
        a = 1 + 2               # Cette instruction ne sera jamais exécutée.
        ```

        ```python showLineNumbers
        # Version 2 – avec validation
        def densite(masse, volume):
            if masse < 0:
                raise ValueError("La masse doit être positive.")
            if volume <= 0:
                raise ValueError("Le volume doit être supérieur à 0.")
            return masse / volume

        print(densite(10, 2))   # ✅ 5.0
        print(densite(-3, 5))   # ❌ ValueError : La masse doit être positive, message clair, le programme arrête ici!
        print(densite(8, 0))    # Cette instruction et les suivantes ne seront jamais exécutées.
        a = 1 + 2
        ```

        👉 La version 2 donne des **messages d'erreur explicites**, évite les résultats absurdes et rend le code **plus robuste et réutilisable**.

        :::

        ---

        :::note `raise` — mettre une barrière dans une fonction
        `raise` permet de **provoquer volontairement une erreur** si une condition n'est pas respectée.\
        C'est la façon de signaler au programmeur qui appelle la fonction qu'il l'utilise avec de mauvaises valeurs.

        ```python showLineNumbers
        def concentration(masse_g, volume_L):
            if masse_g < 0:
                raise ValueError("La masse doit être positive.")
            if volume_L <= 0:
                raise ValueError("Le volume doit être supérieur à 0.")
            return masse_g / volume_L
        ```

        Dès qu'une condition invalide est détectée, Python **interrompt immédiatement** la fonction et remonte l'erreur à l'appelant.\
        Les lignes suivantes à l'intérieur de la fonction ne sont jamais exécutées (comme c'est le cas avec l'instruction `return`).
        :::

        ---

        :::tip `try/except` — mettre un filet de sécurité à l'appel
        `try/except` permet de **rattraper** une erreur qui se produit, plutôt que de laisser le programme planter.

        Exécutez ce code sans `try/except`, observez l'erreur, puis ajoutez le filet et relancez :

        ```python showLineNumbers
        # Sans filet : le programme plante
        print(concentration(4, 20))    # ✅ OK
        print(concentration(-10, 20))  # ❌ Le programme s'arrête ici avec une erreur brute
        print(concentration(4, 5))     # Jamais atteint
        ```

        ```python showLineNumbers
        # Avec filet : le programme gère l'erreur proprement
        try:
            print(concentration(4, 20))    # ✅ Exécuté normalement
            print(concentration(-10, 20))  # ❌ Lève ValueError → on saute au except
            print(concentration(4, 5))     # ⏭️ Jamais atteint (on est déjà dans le except)
        except ValueError as e:
            print(f"Une erreur est survenue : {e}")
        ```

        La variable `e` contient le message d'erreur personnalisé passé à `raise`.
        On peut choisir de faire ce qu'on veut du message :
            - l'afficher à la console
            - le journaliser dans un fichier
            - exécuter un bout de code en particulier en fonction du message
        :::

        ---

        :::note Différence fondamentale entre `raise` et `try/except`

        | Mécanisme | Rôle | Où l'écrire |
        |-----------|------|-------------|
        | <C><K>raise</K></C> | **Signaler** un problème (mettre une barrière) | **Dans** la fonction, là où les paramètres sont vérifiés |
        | <C><K>try</K>/<K>except</K></C> | **Attraper** un problème (mettre un filet) | **À l'appel** de la fonction, pour gérer proprement les erreurs et éviter l'arrêt du programme |

        👉 On utilise `raise` pour protéger la fonction, et `try/except` pour que l'appelant décide quoi faire en cas d'erreur.
        :::

        ---

        ::::info Les fonctions de Python lèvent aussi des exceptions!
        Ce n'est pas seulement votre code qui peut lever des exceptions — les fonctions de la **bibliothèque standard** de Python le font aussi.\
        Comprendre quelles erreurs elles peuvent produire vous aide à les anticiper et même les intercepter (avec un bloc `try/except`) pour éviter l'arrêt pur et simple du programme.

        **`math` — erreurs de domaine mathématique**

        ```python showLineNumbers
        import math

        math.sqrt(25)     # ✅ 5.0
        math.sqrt(-1)     # ❌ ValueError: math domain error  (racine d'un nombre négatif)

        math.log(100)     # ✅ 4.605...
        math.log(0)       # ❌ ValueError: math domain error  (log(0) est indéfini)
        math.log(-5)      # ❌ ValueError: math domain error  (log d'un négatif)

        math.factorial(5) # ✅ 120
        math.factorial(-1)# ❌ ValueError: factorial() not defined for negative values
        math.factorial(2.5)# ❌ TypeError: 'float' object cannot be interpreted as an integer
        ```

        👉 Le module `math` lève une `ValueError` dès qu'on sort du **domaine mathématique valide** (valeur impossible à calculer).

        ---

        **`int()` et `float()` — erreurs de conversion**

        ```python showLineNumbers
        int("42")         # ✅ 42
        int("3.14")       # ❌ ValueError: invalid literal for int() with base 10: '3.14'
        int("abc")        # ❌ ValueError: invalid literal for int() with base 10: 'abc'

        float("3.14")     # ✅ 3.14
        float("abc")      # ❌ ValueError: could not convert string to float: 'abc'
        ```

        👉 `int()` et `float()` lèvent une `ValueError` si la chaîne de caractères **ne représente pas un nombre valide**.\
        C'est une erreur très fréquente quand on lit une saisie utilisateur ou des données provenant d'un fichier.

        ---

        **Intercepter ces erreurs avec `try/except`**

        Voici un exemple concret qui combine `math` et la conversion :

        ```python showLineNumbers
        import math

        def calculer_racine(valeur_texte):
            try:
                nombre = float(valeur_texte)        # Peut lever ValueError si non numérique
                resultat = math.sqrt(nombre)        # Peut lever ValueError si nombre < 0
                return resultat
            except ValueError as e:
                print(f"Erreur : {e}")
                return None

        #Toutes les instructions ci-dessous sont exécutées!
        print(calculer_racine("25"))   # ✅ 5.0
        print(calculer_racine("-4"))   # ✅ La fonction gère l'erreur pour éviter l'arrêt du programme et elle retourne None
        print(calculer_racine("abc"))  # ✅ La fonction gère l'erreur pour éviter l'arrêt du programme et elle retourne None
        a = 1 + 2                      # Cette instruction est finalement exécutée!
        ```

        👉 Un seul bloc `try/except` attrape à la fois les erreurs de conversion **et** les erreurs de domaine, car les deux lèvent une `ValueError`.
        L'appel de la fonction `calculer_racine()` ne causera **jamais l'arrêt du programme** car les erreurs sont bien attrapées.
        La fonction va soit retourner :
            - soit une valeur valide
            - soit la valeur None

        ---

        **Bonne pratique : lire la documentation avant d'utiliser une fonction**

        Les exemples ci-dessus ne sont pas une liste exhaustive — chaque fonction de Python peut avoir son propre comportement en cas d'erreur.\
        Avant d'utiliser une fonction inconnue, **consultez sa documentation** pour savoir quelles exceptions elle peut lever et dans quelles circonstances.

        Par exemple, la page de documentation de [`math.sqrt`](https://docs.python.org/3/library/math.html) indique explicitement qu'une `ValueError` est levée si l'argument est négatif. C'est cette information qui vous permet d'écrire un `try/except` adapté — ni trop large (attraper des erreurs imprévues), ni trop étroit (manquer un cas réel).
        ::::

        ---

        :::tip 🤔 Le réflexe à adopter
        Quand vous intégrez une nouvelle fonction dans votre code, posez-vous toujours la question :\
        **"Qu'est-ce qui pourrait mal se passer ici, et est-ce que ma fonction le gère?"**
        :::

        ---

        :::tip Validation complète avec vérification de type (`isinstance`)
        Pour une validation solide, on peut aussi vérifier que les paramètres sont du **bon type** avant même de vérifier leur valeur.

        ```python showLineNumbers
        def calculer_concentration(masse_g, volume_L):
            if not isinstance(masse_g, (int, float)):
                raise TypeError("La masse doit être un nombre (int ou float).")
            if not isinstance(volume_L, (int, float)):
                raise TypeError("Le volume doit être un nombre (int ou float).")
            if masse_g < 0:
                raise ValueError("La masse doit être positive.")
            if volume_L <= 0:
                raise ValueError("Le volume doit être supérieur à 0.")
            return masse_g / volume_L

        try:
            print(calculer_concentration(10, 2))     # ✅ 5.0
            print(calculer_concentration(-5, 2))     # ❌ Lève ValueError → on saute au except
            print(calculer_concentration(10, 0))     # ⏭️ Jamais atteint (on est déjà dans le except)
            print(calculer_concentration(10, "a"))   # ⏭️ Jamais atteint (on est déjà dans le except)
        except (TypeError, ValueError) as e:
            print(f"Erreur : {e}")
        ```

        La fonction `isinstance(valeur, type)` retourne `True` si la valeur est du type donné.\
        Par exemple : `isinstance(5, int)` → `True`, `isinstance("5", int)` → `False`.\
        On peut tester plusieurs types à la fois : `isinstance(5, (int, float))` → `True`.

        👉 Remarquez l'**ordre des vérifications** : on valide d'abord les **types**, puis les **valeurs**. C'est important, car une valeur du mauvais type (comme une masse de type `str`) ne pourrait même pas être comparée à `0` sans provoquer une erreur.
        :::

        ---

        :::note Exceptions utilisées dans ce cours
        Nous nous limitons aux trois exceptions les plus courantes :

        | Exception | Quand l'utiliser |
        |-----------|-----------------|
        | <C>ZeroDivisionError</C> | Une division par zéro est tentée ou le dénominateur est nul |
        | <C>TypeError</C> | Le type de la valeur transmise n'est pas celui attendu |
        | <C>ValueError</C> | La valeur est du bon type mais ne respecte pas les contraintes (ex. : masse négative) |
        :::

        ---

        :::tip Bonnes pratiques à retenir
        - N'attrapez avec `except` **que les erreurs prévues** — sinon on risque de cacher de vrais bogues.
        - Donnez toujours un **message utile** dans `raise` pour que l'erreur soit facile à comprendre.
        - **Validez d'abord le type**, puis la valeur : une masse de type `str` ne peut même pas être comparée à 0.
        - **L'ordre des validations** dans la fonction a de l'importance : vérifiez ce qui est le plus fondamental en premier.
        :::

        ---

        :::warning Mini checklist de validation
        Avant de retourner un résultat, votre fonction a-t-elle vérifié :
        - ✅ Les types sont corrects (`int`, `float`, etc.) ?
        - ✅ Les domaines sont respectés (positif, non nul, dans un intervalle, etc.) ?
        - ✅ Les cas limites sont couverts (valeur nulle, négative, vide, etc.) ?
        :::

    <TabItem value="match" label="*match/case*">

        :::warning match/case — pour ta culture générale!
        La structure `match/case` produit des résultats très similaires à un `if/elif/else` enchaîné.
        Elle n'est **pas exigée** dans ce cours : tu ne seras **jamais évalué** dessus.\
        Si tu la trouves intuitive, libre à toi de l'utiliser. Un bon vieux `if/elif/else` fera toujours l'affaire!
        :::

        :::info match/case
        Le `match/case` est une structure de contrôle qui permet de faire des correspondances de motifs (*pattern matching*).<br />
        - Il permet de comparer une variable à plusieurs valeurs possibles.<br />
        - Il offre ainsi une alternative aux longues chaînes d'instructions `if`/`elif`/`else` pour des cas spécifiques.

        Sa syntaxe est la suivante :
        ```python showLineNumbers
        match variable:
            case valeur1:
                # code à exécuter si variable == valeur1
            case valeur2:
                # code à exécuter si variable == valeur2
            case _:
                # code à exécuter si aucune des valeurs précédentes ne correspond
        ```
        - Le `_` dans le dernier `case` agit comme un `else`, capturant tous les autres cas non spécifiés.
        - Utile pour des comparaisons simples, mais peut être moins flexible que `if`/`elif`/`else` pour des conditions complexes.
        - Exemple :
        ```python showLineNumbers
        jour = "lundi"
        match jour:
            case "lundi":
                print("Début de la semaine")
            case "mercredi":
                print("Milieu de la semaine")
            case "vendredi":
                print("Fin de la semaine")
            case _:
                print("Jour ordinaire")
        ```
        :::

        :::tip Avantages du match/case
        - Plus lisible pour des comparaisons multiples sur une même variable.
        - Permet de gérer facilement des cas spécifiques.
        - Peut être plus performant que de nombreuses instructions `if`/`elif` dans certains cas.

        👉 Il est important de noter que tout `match/case` peut être réécrit avec des `if`/`elif`/`else`, mais l'inverse n'est pas toujours vrai.
        :::
