# 🍱 Types de données

En Python, chaque valeur a un **type** qui détermine ce qu'on peut faire avec elle : additionner deux nombres, concaténer des textes, itérer sur une liste.

<TabItem value="base" label="Types de base" default>

Chaque valeur que tu manipules (entier, décimal, texte ou booléen) appartient à l'un de ces types.

| Type      | Description            | Exemple            |
|-----------|------------------------|--------------------|
| **int**   | Nombre entier           | <C>x = <N>42</N></C> |
| **float** | Nombre décimal          | <C>y = <N>3.14</N></C> |
| **str**   | Chaîne de caractères    | <C>nom = <S>&#34;Alice&#34;</S></C> |
| **bool**  | Booléen (vrai ou faux)  | <C>actif = <K>True</K></C> |

### Caractéristiques des collections

Choisir la bonne structure dépend de ce dont tu as besoin : ordre, modifiabilité, unicité des éléments ou association clé-valeur.

| Structure | Ordonnée | Modifiable | Doublons permis | Accès | Ajout | Suppression |
|------------|-----------|--------------|------------------|-----------------------------|-----------------|----------------|
| **list**   | ✅ | ✅ | ✅ | Par index <C>[ ]</C> | <C>.<F>append</F>()</C>, <C>.<F>insert</F>()</C> | <C>.<F>remove</F>()</C>, <C>.<F>pop</F>()</C> |
| **tuple**  | ✅ | ❌ | ✅ | Par index <C>[ ]</C> | ❌ (immuable) | ❌ (immuable) |
| **set**    | ❌ | ✅ | ❌ | Test présence <C><K>in</K></C> | <C>.<F>add</F>()</C> | <C>.<F>remove</F>()</C>, <C>.<F>discard</F>()</C> |
| **dict**   | ✅ | ✅ | ❌ (clés uniques) | Par clé <C>d[<S>"clé"</S>]</C> | <C>d[<S>"clé"</S>] = valeur</C> | <C><K>del</K> d[<S>"clé"</S>]</C> |

**Note :** les **dict** conservent l'ordre d'insertion des clés.
