        :::info Fonction simple

        Une fonction est un bloc de code r√©utilisable (un ensemble d'instructions) ayant une t√¢che ou un but sp√©cifique.<br />
        Elle est d√©finie √† l'aide du mot-cl√© `def`, suivi du nom de la fonction, de parenth√®ses et d'un `:`.<br />

        En Python, cette indentation est essentielle pour d√©limiter le bloc de code de la fonction.

        En Python, il existe deux types principaux de port√©e : locale et globale.

        :::info Appels

        Les fonctions peuvent appeler d'autres fonctions.<br />
        Cela est utile pour d√©composer des t√¢ches complexes en sous-t√¢ches plus simples.


        :::

        ---

        :::note Pile d'appels

        Lorsque l‚Äôon d√©marre un programme Python, **une pile d'appels** est automatiquement cr√©√©e. \
        Cette structure de donn√©es, g√©n√©ralement invisible sauf en cas d‚Äôerreur ou via des outils de d√©bogage, \
        permet √† Python de suivre l‚Äôordre d‚Äôex√©cution des fonctions et de conserver l‚Äô√©tat des variables locales.
        
        üîÅ Fonctionnement de la pile

        - Lorsqu‚Äôune instruction **appelle une fonction**, celle-ci est **empil√©e au sommet de la pile d'appels**.
        - Python ex√©cute **une seule fonction √† la fois**, toujours celle qui se trouve **en haut de la pile**.
        - Une fois l‚Äôex√©cution termin√©e, **la fonction est retir√©e de la pile**.
        - **La fonction suivante au sommet** devient alors **la nouvelle fonction active**, que Python ex√©cute imm√©diatement.
        
        üìö Principe LIFO

        La pile d'appels suit le principe LIFO (*Last In, First Out*) :
        - La derni√®re fonction ajout√©e est la premi√®re √† √™tre ex√©cut√©e, et la premi√®re √† √™tre retir√©e.
        - Ce m√©canisme permet √† Python de g√©rer les appels imbriqu√©s de fonctions, et de revenir √† l‚Äô√©tat pr√©c√©dent une fois qu‚Äôune fonction a termin√© son ex√©cution.

        ```python showLineNumbers
        def fonction_a():
            print("Fonction A")
            fonction_b()

        def fonction_b():
            print("Fonction B")

        fonction_a()
        ```

        Dans cet exemple, lorsque `fonction_a` est appel√©e, elle appelle `fonction_b`, qui est ajout√©e √† la pile d'appels.<br />
        Une fois que `fonction_b` a termin√© son ex√©cution, le contr√¥le revient √† `fonction_a`.

        :::

        :::info Commentaires dans le code

        Il est possible d'ajouter des commentaires dans le code pour expliquer ce que fait une fonction ou pour donner des indications sur son utilisation.<br />
        Les commentaires sont ignor√©s par l'interpr√©teur Python et n'affectent pas l'ex√©cution du code.<br />
        Ils sont utiles pour rendre le code plus lisible et compr√©hensible pour les autres d√©veloppeurs (et pour vous-m√™me dans le futur!).

        Pour documenter une fonction, il est possible d'utiliser les triples guillemets `"""` (ou `'''`). \
        Ce type de documentation se nomme un ***docstring*** et est toujours plac√© au d√©but de la d√©finition d'une fonction. \
        Cr√©er une fonction sans mettre une documentation est une mauvaise pratique!
        
        Par exemple :
        ```python showLineNumbers

            Exemple d'utilisation :

            """

            return nb_joute

        # Exemple avec diff√©rentes valeurs

        objet2 = energie_cinetique(5, 7) # Masse de 5 kg, vitesse de 7 m/s

        # Affichage des r√©sultats
        print("√ânergie cin√©tique de l'objet 1 :", objet1, "Joules")
        print("√ânergie cin√©tique de l'objet 2 :", objet2, "Joules")
        ```

        Les lignes 3 √† 19 montrent un exemple de commentaire de bloc utilisant des triples guillemets pour expliquer en d√©tail la fonction `energie_cinetique`.<br />
        Elles sont ignor√©es par Python lors de l'ex√©cution du code.

        Il en est de m√™me pour les lignes 1, 23 et 27 qui sont des commentaires sur une seule ligne, ainsi que tout ce qui se trouve √† droite du `#` √† la ligne 2.
